# Trade-detail page — design

**Status:** design approved 2026-07-06 (brainstormed with Pierre).
**Builds on:** `paper_state.json` open ideas (two-legged: `primary`/`neutralizer`, each with
`ticker`/`direction`/`instrument`/`notional`/`entry_px`, spreads also `S0`/`k_long`/`k_short`/
`T0`/`iv`/`dte`/`debit`), `elp/trades.py` (`idea_return`, `_leg_ret`, `RISK_FREE`),
`elp/options.py` (`bear_put_spread`), `elp/tiingo.py` (`fetch_daily_bars`), `elp/express.py`
(`describe_leg`), and the dashboard/`serve.sh` static-site setup.

## 1. Context and goal

For each open trade, a **Trade details** page (`site/trades.html`, linked from the dashboard) that
shows, per trade: each leg's price over time, the combined trade's return over time, and a table
of the legs and combined — so the reader sees how the return / PV evolved, starting **~1 month
before entry**. Recommendations-only; display of already-computed marks.

## 2. Architecture (keeps `dashboard.py` network-free)

- `elp/tradeviz.py` — pure given bars: series reconstruction (reusing the engine's return math), an
  inline-SVG line-chart generator, and per-trade HTML assembly. No I/O.
- `tradeviz.py` — new entry: reads `paper_state.json`, fetches each leg's `fetch_daily_bars` from
  ~35 days before entry, builds the series, writes `site/trades.html`. Fail-soft. Wired into
  `run_paper.sh` after `dashboard.py`.
- `dashboard.py` — add a top link `Trade details →` to `trades.html` (static link; the file is
  generated by `tradeviz.py`).

Network + charts are isolated in `tradeviz.py`/`elp/tradeviz.py`; `dashboard.py` stays pure.

## 3. Series reconstruction (reuse the engine — do not reinvent)

Per trade, over a date window `[entry − ~35d, last bar]`, aligned on dates common to both legs'
bars:

- **Combined return series:** for each date `d`, build `marks = {p_ticker: px(d), n_ticker: px(d)}`
  and call `elp.trades.idea_return(idea, marks, d)` (after seeding each leg's `_entry_date` from
  `entry_date`). Split at `entry_date`: **solid** from entry to now, **dashed** for `d < entry`
  (the hypothetical earlier hold — `_leg_ret` naturally gives a spread more time value when
  `elapsed < 0`). Combined `P&L$ = ret × primary_notional`.
- **Leg price series** (for the per-leg charts): stock leg → the underlying price `px(d)`; spread
  leg → its **mark** `bear_put_spread(px(d), k_long, k_short, trem(d), iv, RISK_FREE)` where
  `trem(d) = max(T0 − (d − entry).days/365, 1e-6)`.

All fetches fail soft (a leg with no/too-few bars → that trade renders a note instead of charts).

## 4. Charts — hand-rolled inline SVG (no JS, no deps)

`svg_line(series: list[tuple[list, str, bool]], ...) -> str` — each entry is `(points, css_class,
dashed)`; emits one `<svg>` with a `<polyline>` per series, a min/max y-scale, a light baseline,
a vertical **entry marker**, and a tiny legend. Values map to a fixed viewBox (e.g. 640×160);
dates map to x by index. Self-contained, prints/screenshots cleanly, static (no hover/zoom).

## 5. Page layout (`site/trades.html`, all open trades stacked)

Per trade:
- **Header:** `LONG/SHORT <supplier> vs <customer>` · expression · per-leg sizing via
  `describe_leg` (shares/contracts, notional).
- **Leg charts:** one SVG per leg (price/mark over the window) with the entry marker.
- **Combined chart:** trade return % over the window — solid (from entry) + dashed (pre-entry) +
  entry marker.
- **Table:** per leg (ticker, direction, shares/contracts, entry px, latest px, leg P&L $) and the
  combined row (return %, P&L $, current PV/mark). A Grade-C caveat line for spreads.

Empty state (no open trades) and per-trade fail-soft (missing bars) render a plain note.

## 6. Testing (offline, synthetic bars — no network)

`tests/test_tradeviz.py`:
- `svg_line`: given points, output contains `<svg` and a `<polyline` with the expected count of
  coordinates; a dashed series carries `stroke-dasharray`.
- combined-series reconstruction on a synthetic stock-pair (known bars) → the return series matches
  `idea_return` at a couple of dates; the series splits at `entry_date` (dashed before, solid at/after).
- spread-leg mark series uses `bear_put_spread` and decreases time value toward expiry (monotone
  `trem`).
- `trade_detail_html` contains the tickers, the sizing, and the combined return.
- `tradeviz.py` entry: writes `site/trades.html` from a stubbed `fetch_daily_bars`; no open ideas
  or a raising fetch → still writes a page, exits 0 (fail-soft).

## 7. Out of scope
- Interactive charts (hover/zoom/JS); a charting dependency.
- Closed-trade history pages (open trades only for now).
- A precise two-leg option cost/greeks model (spread PV stays Grade-C, flat IV — labeled).
- Changing the engine, `paper_state.json` shape, or how trades open/close.
